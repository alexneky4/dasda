1.Introduction

1.1.Purpose

EmoF is a site that helps people to get constructive feedback on a product/service that they want to improve. The constructive feedback consists of emotions from the Plutchik model that the users feel when they see the product/service. The website will be able to provide statistics on each form after it has ended.

1.2.Product Scope

The site's purpose is to help people get constructive feedback on a product/service that they want to improve. There are many people in the world that want to implement or create something inovative, but don't know exactly all of the needs of the humans. Constructive feedback is the best way to learn this. Our software comes in help with an unique feature: the possibility to choose an emotion that an user felt when seeing the product and its characteristics. The user will also have the possibility to explain why he felt that way. Based on these emotions, when the form ends, you can see the statistics of interest, so the owner can see which is the general opinion(and what can improve) and an user can see if it's recommended at that moment of time or not.

The website will also have an API that can be used by other applications to communicate and get data from the application. The data can also be able to be exported in a HTML, CSV and JSON file.

2.Overall Description

2.1.Product Perspective

EmoF is a new, self-contained product with a few things in common with Google Forms, based on emotion feedback.

2.2.Product Functions

Functionalities of the site:

  • Being able to create and log into an account on the platform;
  • Sending anonimous feedback for a product/service or concept;
  • Creating and uploading a form with the product/service you want to receive feedback (only if you have an account);
  • Filter the forms by some common tags or by name
  • Sort forms based on expiration date and popularity
  • Having in your profile a list with forms that you uploaded;
  • Visualising statistics with the feedback received on the forms with the emotions received;
  • The site with an API REST/GraphQL will manage the forms and all the responses;
  • Being able to export the data in a HTML, CSV and JSON files;
  • Having the possibility to manage forms inside the web application as an administrator;

2.3.User Classes and Characteristics

There are 3 possible types of users which can use the website:

  1. Users that create and upload a form for feedback
    • they have something/someone that can(and want) to make better;
    • they want to hear the opinion of people in order to see what it's good and what not in the object from the form;
    • they want to see what is the most predominant emotion that is triggered by the object;
    • they also want to ask pertinent questions in order to see what can be improved;
    • the object can be from a variety of categories(some examples are listed below):
      • a person
        • it can be the user that uploads the form
        • usefull to see how good he is at his/her profession(for example: public speaker, singer, doctor)
      • a geographical place
        • see if the place needs renovation(example: a castle)
        • see how safe it is to walk there(especially for places in the nature, like caves or mountain trails)
      • a product
        • see if the product fulfills the requirements for which it was created
        • see if it raises to the expectations of the public
        • see what updates(if are any) can be added
      • a hospital
        • see if the sanitory conditions are met
        • see if the doctors do the job right
  2. Users that give feedback
    • they want to say their opinion about an object from the site;
    • they give constructive feedback in order to help;
    • they can select a certain category of forms from where to complete one or more;
    • in the form they also select an emotion;
  3. Users that see the statistics
    • they want to see the opinion of people in order to know what it's recommended and what it's not;
    • they can discover different elements from a certain category;
    • they can view the statistics in HTML, CSV or JSON formats;

Note: an user can be part of more categories(for example, he can create forms and give feedback to other forms).

2.4.Operating Environment

The programming languages used for the front-end are HTML, CSS and JavaScript to communicate with the back-end and help design pages, structure them and make them responsive, such that they can be accessed from any device. PHP will be used for the back-end part, together with MySQLi extension.

2.5.Design and Implementation Constraints

In order to open the CSV format of the statistics, you need to have access to Google Sheets or a text editor(or it can be opened with Excel). For the JSON format you need to have an open-source text editor on your device(for example, File Viewer for Android). Both of these formats can also be opened from the browser(with an online viewer).

3.External Interface Requirements

3.1.User Interfaces

All pages from this website have in common the same header that contains the title of the website and links to other pages from the site.

When you first enter the site, you are redirected to the start page. It contains the title of the application, a short description and the flower with Plutchik model. There is also a "Start now" button used to continue without making an account.

In "Register" page, you have the possibility to create an account(if you already have one, just click on "Already have an account?" phrase). There are 6 fields that you need to complete: "Username", "Password", "Email", "Date of birth", "Gender" and "Country". After you complete all of the fields, you can press the "Register" button. Each field requires a specific data(only some characters can be written, the data needs to respect a format), so if at least one of them doesn't have the required information(or it is empty), you won't be registered, and the fields which are not correctly completed will have red margins. You will also see in these fields an error message, depending on what you didn't respect.

"Login" page is used to connect to the site with your account, so that you can have access to your data. The user name and password are required in order to login, so make sure that you don't forget your password(or write it somewhere safe).

In the "Home" page, you have at the left of the page a field with your photo and your name (if you don't have an account these are empty). Below, there are checkboxes with different types used to filter forms based on different categories. The rest of the page has a search bar to write the name of the thing you want to see and forms below, with rectangular form, containing inside a photo with the element, name, time left for completion and a short description. Pressing on any form will open a new page used to give feedback for that form.

The page used to give feedback contains at the top the title of the object, then some images with the object and optionally the main description. Then you have an area where you can select the emotion overall for the object and a short description. Then there will be areas for each component, that contain the component title and optionally a short description and a box from where you select an emotion for the overall product and a text area where you can write a short description why you felt that way if you wish to. At the end you can submit using the "Submit" button.

"Create" page has the option to make a form. There are many elements to be completed including the form name, description, main-image for the form, the expiration date, if the statistics for the form should be public and if the form should still be seen after it expires. Optionally you can add more images to the form and add components to the form. For each component, you can add a title, description and images. At the end of the page, there is a submit button that will verify the data and if the data passes the verification then it will redirect to a success page.

In the "About" page, you can find some details and information related to the site and the mission of it. Here is also implemented a link which leads to this documentation.

The "Profile" page has stored the information about you in the left part. Besides what you completed at the register part, you have the number of forms that you created, the number of forms that you completed and a photo(if you want to upload one). The rest of the page contains the forms where you completed at least one answer. You have the name of the form, the date when you started to write in the form and how much time do you have to complete it until the form closes. There is also a progress bar, which shows how much of the form is completed and an equivalent percentage.

From the profile and forms, you will also be able to view statistics about each form. The page starts with the title of the form, the time when it expired and the description. Then contains the statistics about the feedback given by the users showing different graphs and charts and the reviews given.

The "Statistics" page displays all forms you can see the statistics of and a search bar for searching for forms. After pressing on a form, you will be redirected to the "give-statistics" page.

The "Give Statistics" page displays for a form the statistics for the responses. Here you can choose the components you wish to see the statistics of and the age range of the users and the time elapsed since they gave the feedback. After choosing from these settings, you can press check and it will display charts with the emotions chosen by the users. Then for each chart, you can choose to view the responses and it will show a table with the users that responded and the emotions given. This data can be exported in JSON and CSV for each chart.

3.2.Hardware Interfaces

The website can be accessed from any computer, laptop or mobile device. The communication protocol which is used is HTTP.

3.3.Software Interfaces

In order to save information about each profile of the users and forms that they uploaded, we'll use a database. The database has multiple tables: one for storage of the accounts, one for the forms from the site and one for the statistics. The username has the role of primary key in the accounts table, so there won't be two users with the same username. This field is case sensitive. In the forms table, each form has a number which is uniquely associated with an username(the user that created the form) and an unique id number used to identify the corresponding statistic from the third table. There will also be tables storing information about the forms and the feedback given.

3.4.Communication Interfaces

The HTTP protocol is used for data transfer.

4.System Features

4.1.Account Creation

4.1.1.Description and Priority

You need to create an account in order to upload forms on the website. This feature is available in the "Register" page and has a high priority level.

4.1.2.Stimulus/Response Sequences

The user goes to the "Register" page, where he needs to complete 6 fields: "Username", "Password", "Email", "Date of birth", "Gender" and "Country". After all of the fields are completed, the button "Register" will be pressed. If no errors occurs, the user will be sent to the "Home" page and the information used will be available in the "Profile" section. If the person that access the website already has an account, he can press "Already have an account?" section from the "Register" or he can go directly to the "Login". The only things required are "Username" and "Password". People can access the site and complete forms without having an account, but they won't be able to create and upload forms.

4.1.3.Functional Requirements

Each field has a specific type which needs to be respected. "Username" field can have letters, digits, spaces and '_' character, but the first character cannot be a digit or space. "Password" field can have any character from the QWERTY(see image below) layout except space and the characters written will be printed on the screen as a dot. "Email" field needs to respect the format 'name@platform.com'. "Date of birth" field is of type date and you can select a date from the calendar(icon in the right of the field). "Gender" field has a selection menu and you can choose from "Male", "Female" or "Other". "Country" field can have any country from the world. If any of these fields are incorrectly completed(or are not completed), the profile won't be created, and the fields which don't correspond with the format will have red borders and an error message inside the field. The "Username" is unique, where won't be two profiles with the same name, so if the name chosen already exists there will be an error and the user will have to think of another username. All of these data will be stored in a database with "Username" as a primary key and can be visible for the user in the "Profile" page.

4.2.Sending Feedback

4.2.1.Description and Priority

Any person that access the site can select one or more forms to complete, giving feedback to the form owner. This feature is available at the "Home" page and has a high priority level.

4.2.2.Stimulus/Response Sequences

The user goes to the "Home" page and selects a form which is not closed(forms which have expired can remain on the site). Here, he has a page with the title, images and a short description. After that, there is a selection menu, from which the user choose an emotion for the overall object. Then, each component(defined by the owner) has a selection menu with emotions and a field for writing what part of the object made you feel like the emotion you selected. After you complete all of the components, you can press the "Submit" button. Then you are redirected to the "Home" page. In the left, there are multiple boxes, each one representing a category of object from forms. The user can click on one or more boxes, so that he will have only forms which corresponds with these categories. There is also a search bar, so that the user can search a specific object, no matter the category.

4.2.3.Functional Requirements

Each form is open for a limited time, after that it can still be on the site, if the person making the form chose to do so, but it would be closed. If a person doesn't complete the entire form until it closes, his feedback won't be taken in consideration. The form has a number of components. You need to select an emotion for the overall object and optionally an emotion for each component of the object. If you don't complete any of the mandatory fields, when you press "Submit", the website will tell you what is wrong and you won't be able to submit. For the text part, each answer is optionally if the person giving feedback wants to give more information about why he felt that way.

4.3.Upload a Form

4.3.1.Description and Priority

If you have an account, you can create a form where you can add images and components. This can be done at the "Create" page and has a high priority level.

4.3.2.Stimulus/Response Sequences

In the create page, you have a field for the form name, a field for description, fields for components of the object(2 by default, if you want to add more press enter when you are in the last field), a field where you set an expiration date, the question "Should the form still be available for the public after the expiration date?", where you need to select "Yes" or "No", and a field for uploading images. Except components and images, all of these are mandatory to be completed. After that, you can press "Create new form" button.

4.3.3.Functional Requirements

You cannot create the form if a field(except components and images) is not completed. The form name needs to have at least 5 characters, the description needs to have at least 20 characters, expiration date needs to be set in the future and for the question about what happens to the form when it expires you need to select "Yes" or "No". If at least one of these is not completed when you press "Create new form", the website will tell you what is wrong and you will have to fix the errors for it to work.

4.4.Visualise Statistics

4.4.1.Description and Priority

Any user can see the statistics based on emotions for the objects whose forms expired. This can be done at the view feedback page accessed from the forms or profile and has a high priority level.

4.4.2.Stimulus/Response Sequences

The user goes to the view feedback page, where he has available four rectangles: one with data about the form(name, time when ended, description and number of feedbacks), one with responses to each question, one with a pie chart representing how much percentage was for each emotion(the emotions are written in the right of the chart) and one with a rectangle for each emotion. The rectangles have the length directly proportional with the number of votes for that emotion. These statistics can also be available in CSV and JSON format.

4.4.3.Functional Requirements

By default, the visualisation will be on HTML format. If you select CSV or JSON format, a file will automatically download to your device.

5.Other Nonfunctional Requirements

5.1.Performance Requirements

There are no performance requirements for using the application(only to have internet connection and a browser).

5.2.Security Requirements

For the security part, the server can be behind a firewall. For login, we can make sure that no SQL injection can be done. A DDoS mitigation service can also be implemented.
